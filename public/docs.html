PaperBanana REST API

Generate academic diagrams programmatically. Same pipeline and styling options as the web UI — exposed over bearer-token-authenticated HTTP.

Base URL: https://paperbanana-generator.vizuara.ai/v1

Authentication

Every request must include an Authorization header:

Authorization: Bearer pb_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

Keys are minted by the workspace owner. Contact Vizuara if you need one. Keys are shown in plaintext exactly once at creation; only a SHA-256 hash is stored server-side.

Rate limits

Default: 100 generations per UTC day per key. The quota is set per-key at creation time. When exceeded, the API returns 429 quota_exceeded.

Endpoints

POST /v1/generate — streaming sync

Runs the full pipeline and streams progress as application/x-ndjson (one JSON object per line). Takes 60–90 s for the default 3-iteration run.

Request body

FieldTypeRequiredDescription
descriptionstring✓Natural-language description of the figure.
iterationsint 1–3—Pipeline iterations (default 3).
stylestring—Style preset (see GET /v1/config).
text_densitystring—One of standard, minimal, light, dense.
color_themestring—Palette override (see config).
aspect_ratiostring—One of 1:1, 2:3, 3:2, 3:4, 4:3, 9:16, 16:9, 21:9.
modelstring—auto (default) or an explicit image model id.

Stream events

Each line is a JSON object with a type field:

  • meta — echoes the resolved parameters
  • gen:start / gen:done — per-iteration image generation
  • critic:start / critic:done — per-iteration critique
  • done — pipeline complete, with totalMs
  • error — pipeline failed

Images arrive as base64 under image.base64 with image.mimeType.

POST /v1/jobs — async

Returns { job_id, status_url } immediately; pipeline runs in the background.

GET /v1/jobs/:id — poll

Returns current job state. Poll every few seconds. Completed images arrive inline as base64.

GET /v1/jobs — list your key's recent jobs (last 24 h, up to 50)

POST /v1/overlay — composite a heading / logo / URL

Takes a image_base64 (typically one of your generated iterations) plus optional heading, show_logo, url. Returns a new PNG.

GET /v1/config — capabilities discovery

No auth required. Returns the full list of styles, text densities, color themes, aspect ratios, image models.

Examples

curl — sync streaming

curl -N -X POST https://paperbanana-generator.vizuara.ai/v1/generate \
  -H "Authorization: Bearer $PB_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "description": "A left-to-right transformer pipeline with tokenizer, positional encoding, encoder blocks, decoder with masked attention, and softmax output.",
    "iterations": 3,
    "style": "research-paper",
    "color_theme": "ocean",
    "aspect_ratio": "16:9"
  }'

curl — async

# 1. create the job
curl -X POST https://paperbanana-generator.vizuara.ai/v1/jobs \
  -H "Authorization: Bearer $PB_KEY" \
  -H "Content-Type: application/json" \
  -d '{"description":"U-Net segmentation architecture", "iterations": 3}'
# → {"job_id":"job_abc...","status_url":"..."}

# 2. poll until status == "succeeded"
curl https://paperbanana-generator.vizuara.ai/v1/jobs/job_abc... \
  -H "Authorization: Bearer $PB_KEY"

Python — async with polling

import os, time, base64, requests

BASE = "https://paperbanana-generator.vizuara.ai/v1"
H = {"Authorization": f"Bearer {os.environ['PB_KEY']}"}

r = requests.post(f"{BASE}/jobs", headers=H, json={
    "description": "Retrieval-augmented generation pipeline with a vector store.",
    "iterations": 3, "style": "presentation", "color_theme": "berry",
})
r.raise_for_status()
job_id = r.json()["job_id"]

while True:
    job = requests.get(f"{BASE}/jobs/{job_id}", headers=H).json()
    if job["status"] in ("succeeded", "failed"): break
    print(job["progress"]); time.sleep(3)

assert job["status"] == "succeeded", job["error"]
for it in job["iterations"]:
    with open(f"iter-{it['iteration']}.png", "wb") as f:
        f.write(base64.b64decode(it["base64"]))
print(f"wrote {len(job['iterations'])} images")

Node — streaming sync

import fs from "node:fs";

const res = await fetch("https://paperbanana-generator.vizuara.ai/v1/generate", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${process.env.PB_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ description: "...", iterations: 3 }),
});
const reader = res.body.getReader();
const decoder = new TextDecoder();
let buf = "";
while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  buf += decoder.decode(value, { stream: true });
  const lines = buf.split("\n"); buf = lines.pop();
  for (const line of lines) {
    if (!line.trim()) continue;
    const ev = JSON.parse(line);
    if (ev.type === "gen:done") {
      fs.writeFileSync(`iter-${ev.iteration}.png`,
        Buffer.from(ev.image.base64, "base64"));
      console.log(`saved iteration ${ev.iteration}`);
    }
  }
}

Error shape

{
  "error": {
    "code": "quota_exceeded",
    "message": "Daily quota of 100 requests reached. Resets at 00:00 UTC.",
    "used": 100,
    "quota": 100
  }
}

Common codes: unauthenticated, quota_exceeded, invalid_request, not_found, overlay_failed, pipeline_error.

Admin (for the workspace owner)

Gated by the server's ADMIN_TOKEN env var.

# Mint a key
curl -X POST https://paperbanana-generator.vizuara.ai/v1/admin/keys \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"Rajat - iPad tests", "daily_quota": 200}'
# → { id, name, daily_quota, key: "pb_live_..." }   ⚠ save `key` NOW

# List keys + today's usage
curl -H "Authorization: Bearer $ADMIN_TOKEN" \
  https://paperbanana-generator.vizuara.ai/v1/admin/keys

# Revoke
curl -X DELETE -H "Authorization: Bearer $ADMIN_TOKEN" \
  https://paperbanana-generator.vizuara.ai/v1/admin/keys/key_xxxxxxxx
